### Базовая, минимальная автоматизация настройки свежей VPS.
</div>

### 🔗 VPS Setup
```bash
bash <(curl -Ls https://raw.githubusercontent.com/CeltMain/VPSSetup/refs/heads/main/vpssetup.sh)
```

---

<div align="center">

### 🛠 Основные возможности автоматизации:

</div>

* **Кастомный SSH**: Позволяет установить любой порт SSH на выбор или сгенерировать случайный безопасный порт.
* **Гибкий Swap**: Автоматически сканирует память и предлагает создать чистый файл подкачки настраиваемого размера `[Default: 2GB]`.
* **Безопасный DNS**: Настройка шифрования трафика DNS over TLS с умным детектом интерфейсов `[Default: Cloudflare]`.
* **Защита UFW**: Конфигурация файрвола с автоматическим открытием портов веб-сервера (443/tcp) и выбранного SSH.
* **Фильтрация IPv6**: Опциональное отключение создания и обработки правил для IPv6 на уровне файрвола `[Default: n]`.
* **Скрытый режим (NoPing)**: Полное отключение ответов на входящие ICMP-запросы (пинг) для защиты VPS от сканирования.
* **Патчи безопасности**: Включение системной службы автообновления и бесшумной установки критических обновлений ОС.

---

<div align="center">

### 🔒 Архитектура безопасности и изоляции:

</div>

* **Бесшумное создание**: Автоматически создает системного пользователя `sshtunneluser` в скрытом режиме без лишних анкетных данных.
* **Блокировка паролей**: Намертво отключает возможность авторизации по паролю для защиты туннеля от брутфорса `[Passwordless]`.
* **Двойной режим**: Разрешает **одновременное** использование сетевых SSH-туннелей (TCP Forwarding) и передачу файлов по SFTP.
* **Строгий Chroot**: Жестко изолирует сессию пользователя внутри домашней папки — он физически не видит системные файлы VPS.
* **Встроенный SFTP**: Конфигурирует безопасный изолированный модуль ядра `internal-sftp` с заблокированным терминалом.
* **Область записи**: Автоматически размечает доступную для записи папку `Download` внутри закрытого Chroot-окружения.
* **Валидация синтаксиса**: Выполняет принудительную проверку конфигурации демона SSH перед рестартом во избежание сбоев сети.

<div align="center">

  ### 🧹 Завершающий этап выполнения

</div>

- **APT Cleanup**: Удаление изолированных зависимостей, очистка кэша пакетов и устаревших архивов загрузок.
- **Docker Prune**: Полная зачистка неиспользуемых контейнеров, висящих слоев и кэша Docker-образов.
- **Journald Vacuum**: Безопасное принудительное сжатие раздувшихся системных журналов логирования до лимита в 100 МБ.
- **Logrotate Policy**: Автоматическое создание изолированных правил ежедневной ротации syslog с ограничением веса файлов до 50 МБ.
